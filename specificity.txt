Paragraph A has the following properties: color(gray), font size(1rem), font family(georgia, sarif), line height(1.4), padding(4px 0) and they only come from element selector p.
Paragraph B matches both the p element selector and the .info class selector. Because class selectors have higher specificity than element selectors, the properties defined in .info override those from p where they overlap. So the paragraph has the following properties: color(steelblue), font size(1.25rem), font family(arial, sans-serif), background color(#e8f4fd), padding(8px 14px), border left(5px solid steelblue), line height(1.4).
ragraph C matches the p element selector and the #warning id selector. Since id selectors have higher specificity than both class and element selectors, the properties from #warning override those from p. The final properties of the paragraph C are: color(darkred), font-size(0.8rem), font-family('Courier New', monospace), font-style(italic), text-transform(uppercase), letter-spacing(0.15em), border(2px dashed darkred), padding(10px), background-color(#fff0f0), line-height(1.4).
Paragraph D matches all four selectors: p, .info, #warning, and #warning.info. Among these, #warning.info has the highest specificity because it combines an id and a class. Therefore, for properties defined in this rule, the browser uses the values from #warning.info. Final properties for paragraph D: color(darkorange), font family('Courier New', monospace), font size(1.1rem), background color(#fff7e0), border(2px solid darkorange), font style(normal), letter spacing(0.03em), padding(10px).
The browser uses specificity: id > class > element. Combined selectors like #warning.info are more specific than a single id. If the most specific rule doesn’t define a property, the browser uses the next most specific rule that does. Color: darkorange, because #warning.info has the highest specificity. Font-family: 'Courier New', monospace, because #warning.info does NOT define it, so it falls back to #warning.
